From Single Agent To AI Team

把编码 CLI 升级成多智能体团队模式

OMO、OMC、OMX 都是在现有编码工具之上加一套编排层：主 Agent 常驻做总指挥，遇到专业任务就动态生成独立子 Agent，并行、异步、隔离地完成工作，结束后自动回收。

01 / Architecture

主 Agent 是总指挥，不是搬砖工

OMO / OMC 里常见的主控是 Sisyphus；OMX 则把 Codex CLI 通过 AGENTS.md、prompts、skills 和 team 命令组织成主控入口。它们共同负责理解、拆解、调度、验收和持续推进 TODO。

编排层Sisyphus / Codex 主控常驻或入口式主 Agent，判断任务类型，决定该派谁上，负责最终整合。
规划层Prometheus / Metis / Momus做需求澄清、计划生成、找缺口、审计划。
专家层Oracle / Librarian / Explore处理架构、文档、代码探索等高专业度或高上下文任务。
执行层Worker / Frontend / Tester真正改代码、写页面、补测试、产出可合并结果。

02 / Dynamic Spawn

子 Agent 自动起来，核心是四步闭环

从用户的一句话，到多个后台 Agent 并行跑完，这类工具依赖的是意图识别、动态生成、隔离执行、结果回调这一整条链路。OMX 的入口更偏 Codex CLI：常用 $team、$ralph 或 omx team 来触发团队化执行。

意图识别

Sisyphus 先过 IntentGate：查文档给 Librarian，架构评审给 Oracle，写前端给 Frontend Engineer。

动态生成

调用内部 Agent 生成接口，创建独立会话、独立上下文、专属模型和最小必要上下文。

/subagents spawn oracle · omx team

独立执行

子 Agent 使用自己的模型和工具链运行，主 Agent 不被阻塞，可以继续调度其他任务。

回调回收

完成后返回代码、文档或结论。Sisyphus 校验整合，子 Agent 自动销毁或休眠。

03 / Why It Works

能跑起来，靠的是隔离、路由、Hook 和工具链

表面看是“叫几个 Agent 干活”，底层实际是会话、上下文、模型、工具权限和生命周期管理的组合。

异步非阻塞

  • 后台异步任务调度
  • 独立会话或运行时状态管理
  • 多个子 Agent 同时跑，主 Agent 继续推进

三级上下文

  • 项目规则：AGENTS.md
  • 主/子 Agent 会话上下文隔离
  • 子任务临时上下文或 .omx/ 运行态，用完即清

多模型路由

  • Sisyphus：强推理主脑
  • Oracle：逻辑和架构
  • Librarian：文档和长文本
  • Explore：快速扫代码

生命周期 Hook

任务前、任务中、任务后都能插入自定义逻辑，比如权限检查、日志、验收、格式化、通知。

MCP 工具链

内置 MCP、项目 .mcp.json、Skill 内嵌工具三层组合，子 Agent 按需拿工具，不把工具堆给所有人。

用完即散

子 Agent 不常驻。结果回传后释放资源，只保留主 Agent 的稳定上下文和最终结论。

OMX 运行时

OMX 参考实现里还有 .omx/ 状态目录、HUD、prompts、skills 和 hooks，用来把 Codex CLI 包成可编排团队。

04 / Example

例子：给 Next.js 加登录认证

Input

用户输入：ulw / ultrapilot / omx team 给 Next.js 加登录认证。

Route

主 Agent 判断需要需求澄清、架构、前端、后端和测试。

Plan

Prometheus 访谈和查代码，Oracle 评审方案并选择技术栈。

Build

Frontend Engineer 写登录页，后端 Agent 写 API、JWT 和数据库逻辑。

Verify

Tester 写单元测试和集成测试，多个 Agent 并行后台跑。

Finish

主 Agent 汇总代码、解决冲突、收尾，子 Agent 全部回收。

一句话总结

OMO、OMC、OMX 都是在把单人编码助手升级成“AI 团队模式”：主 Agent 当老板，动态召唤各路专家子 Agent 并行干活，干完就散，高效又省钱。

06 / Decision

怎么选？看你平时用什么

核心体验都是“主 Agent + 动态子 Agent 团队”。选择时不用纠结架构差异，优先跟随你当前的日常工具栈。

你用 OpenCode

装 Oh My OpenCode。它原生贴合 OpenCode 的终端、文件、LSP 和开放模型生态。

你用 Claude Code

装 Oh My Claude Code。它对 Claude Code 的交互和记忆体系更顺手。

你用 Codex CLI

看 Oh My Codex。它把 AGENTS.md、prompts、skills、MCP、hooks 和 team 命令整合到 Codex 工作流里。

多个都用

随便从最常用的入口开始。多 Agent 编排的核心体验一致，迁移成本主要在命令名和工具链细节。
