Mac Setup安装 Go

第一步：在 macOS 安装 Golang

方式一：官网下载，最稳
  • 打开 go.dev/dl。
  • 下载 macOS 安装包：Apple Silicon 选 arm64，Intel 选 amd64。
  • 一路安装，默认会放到 /usr/local/go。
  • 安装后重新打开终端。
方式二：Homebrew，适合开发机 
brew update
brew install go

go version

看到 go version go1.xx darwin/arm64 就说明安装成功。

先确认 go 命令能跑，再开始写代码2 / 24
讲法：如果公司机器已有 Homebrew，用 brew 最快；如果环境比较受控，用官网 pkg 更直观。重点不是安装方式，而是最后 go version 能看到版本。
Mac Setup环境变量

第二步：设置 Go 环境变量

配置环境变量 vi ~/.zshrc 

# 这里改成你的 go 程序目录
export GOROOT="/usr/local/Cellar/go/1.26.3"
export GOPATH="$HOME/go"
export GOBIN="$GOPATH/bin"
export PATH="$GOROOT/bin:$GOBIN:$PATH"
其它：命令行下执行 
go env -w GOPROXY=https://goproxy.cn,direct
go env -w GO111MODULE=auto

# 验证配置
go env GOROOT GOPATH GOBIN
go env GOPROXY GO111MODULE

保存后执行 source ~/.zshrc，或者新开一个终端窗口也可以。

环境变量配好后，go run、go install 才不容易踩坑3 / 24
讲法：这一页按用户实际操作讲。先 vi ~/.zshrc，把 GOROOT/GOPATH/GOBIN/PATH 写进去；source ~/.zshrc 或新开终端。GOPROXY 和 GO111MODULE 用 go env -w 在命令行执行。

Why Go先建立直觉

Go 常被用来写什么

  • Web API 服务：登录、订单、配置、后台管理接口
  • 微服务：拆成多个小服务，各自处理一类业务
  • 工具和脚本：命令行、构建工具、数据处理
  • 高并发任务：网关、推送、消息消费
给前端的理解

如果 Node.js 像“灵活的胶水”，Go 更像“规矩清晰的工程语言”。它编译快、部署简单、运行稳定。

关键词：简单、稳定、容易部署5 / 24
可以补一句：Go 的语法没有 JavaScript 灵活，但团队协作时代码风格更统一。

Variables变量和类型

Go 是静态类型，但写起来不重

var name string = "Ada"
age := 28
price := 19.9
ok := true

fmt.Println(name, age, price, ok)
记住三点
  • var 可以显式写类型。
  • := 让 Go 自动推断类型。
  • 类型一旦确定，就不能随便变。
类比：TypeScript 里 let age: number9 / 24
这里用 TS 类比：Go 更像默认开启严格模式的 TS。
Function函数

函数：参数写类型，返回值也写类型

func add(a int, b int) int {
    return a + b
}

func hello(name string) string {
    return "Hello, " + name
}
读法

func add(a int, b int) int

定义一个 add 函数，接收两个 int，返回一个 int。

Go 函数声明比 JS 更规整10 / 24
不要讲闭包、高阶函数太多。先保证他们能读懂普通函数。
Struct数据结构

struct：Go 里的“数据模型”

type User struct {
    ID   int
    Name string
    Role string
}

user := User{
    ID: 1,
    Name: "Ada",
    Role: "admin",
}
类比 TypeScript 
type User = {
  id: number
  name: string
  role: string
}
字段首字母大写通常表示可被外部访问11 / 24
这一页很关键：后面返回 JSON 就靠 struct。顺便点一下大写导出，小写私有。
Collections数组感

前端最常见：slice 和 map

slice：像数组

names := []string{"Ada", "Lin"}
names = append(names, "Ken")

map：像对象字典

scores := map[string]int{
    "Ada": 95,
    "Lin": 88,
}

先记：列表用 slice，键值对用 map。

日常 API 开发已经够用12 / 24
不要深入数组和 slice 底层差异，只讲日常怎么用。
Error错误处理

Go 不太用 try/catch，错误要显式判断

data, err := os.ReadFile("config.json")
if err != nil {
    fmt.Println("read failed:", err)
    return
}

fmt.Println(string(data))
前端同学要适应
  • err != nil 很常见。
  • 错误处理写得明显，代码更直白。
  • 看到 err 先问：失败时该怎么办。
nil 可以理解为空值13 / 24
告诉大家不要嫌重复。Go 的哲学是把失败路径摆在明面上。
HTTP Server实战开始

用标准库写一个 HTTP 服务

package main

import (
    "fmt"
    "net/http"
)

func main() {
    http.HandleFunc("/ping", func(w http.ResponseWriter, r *http.Request) {
        fmt.Fprintln(w, "pong")
    })

    http.ListenAndServe(":8080", nil)
}
浏览器打开：http://localhost:8080/ping14 / 24
这一页建议现场运行。强调 Go 标准库就能写 HTTP，不一定一上来用框架。
JSON API前端最关心

返回 JSON：前端就能 fetch 了

import (
    "encoding/json"
    "net/http"
)

type User struct {
    ID   int    `json:"id"`
    Name string `json:"name"`
}

func userHandler(w http.ResponseWriter, r *http.Request) {
    w.Header().Set("Content-Type", "application/json")
    user := User{ID: 1, Name: "Ada"}
    json.NewEncoder(w).Encode(user)
}
结构体字段大写，json tag 控制输出字段名15 / 24
解释反引号里的 json tag：Go 字段叫 Name，返回给前端是 name。
Frontend Fetch连起来

前端调用它，和平时一样

fetch("http://localhost:8080/user")
  .then(res => res.json())
  .then(user => {
    console.log(user.id, user.name)
  })
你要对齐的接口信息
  • URL 是什么。
  • 请求方法：GET、POST。
  • 请求体字段。
  • 响应 JSON 字段。
  • 错误码和错误格式。
API 开发的核心是契约，不是语言16 / 24
把前后端协作拉回来：Go 只是实现方式，接口契约才是共同语言。
One File Demo完整示例

一个最小可跑版本

package main

import (
    "encoding/json"
    "net/http"
)

type User struct {
    ID   int    `json:"id"`
    Name string `json:"name"`
}

func main() {
    http.HandleFunc("/user", func(w http.ResponseWriter, r *http.Request) {
        w.Header().Set("Content-Type", "application/json")
        json.NewEncoder(w).Encode(User{ID: 1, Name: "Ada"})
    })

    http.ListenAndServe(":8080", nil)
}

Project Tour真实项目

inke.activity.service：先看它是什么

项目定位

这是一个活动服务项目：承接活动 H5 的接口、后台配置接口、活动配置加载、定时任务和消息消费。

前端同学可以把它理解成：很多活动页面背后的 API 服务集合。

入口路径
  • app/main/service_api/main.go：API 进程入口。
  • server/http/http.go：初始化 HTTP 服务。
  • server/http/router.go：全局路由分发。
  • act/：每个活动自己的业务代码。
真实项目不要先钻细节，先找到入口链路20 / 24
讲法：先告诉大家这个项目不是“一个活动”，而是很多活动共用的后端服务。读项目先找 main、http、router、act。
Structure目录结构

常用目录：知道去哪里改

act/

活动代码主目录，通常按年份、月份、活动名分层。每个活动会有 route、service、model、backend 等文件。

app/config/

服务配置和活动 JSON 配置。不同环境会有不同目录，比如 ali-testqa、ali-bj。

server/http/

HTTP 路由入口。新活动写好后，通常要在这里挂到全局路由。

app/main/

不同进程入口：service_api 负责接口，service_cron_* 负责定时或消费类任务。

dao / manager

数据库访问、Redis、下游服务封装。不要把所有逻辑都堆在 handler 里。

docs/

活动开发规范和场景手册。新需求先看文档，再决定复用哪个模板。

目录不是死记硬背，是为了快速定位修改点21 / 24
强调：前端同学参与联调时，最常用的是 act、app/config、server/http 三块。
Config配置与 H5

配置分两类：服务配置和活动配置

服务配置 TOML

app/config/service-api.toml
app/config/ali-testqa/service-api.toml
app/config/ali-bj/service-api.toml

关键字段：
act_config       # 活动 JSON 根目录
api_domain_name  # API 域名
h5_domain_name   # H5 页面域名

活动配置 JSON

app/config/ali-bj/activitys/
  2026/
    act202606/
      xxx_activity.json

常见内容：
活动时间、奖励、开关、H5 URL、文案、资源配置
前端路由通常不在本仓库，更多体现为 H5 链接配置22 / 24
讲清楚：这个仓库主要是后端服务。所谓前端路由/页面地址，通常在活动 JSON 里的 h5_url/url，或者由 h5_domain_name 拼接。
Routes前后台路由

路由链路：从 main 到活动接口

1 MAINservice_api/main.go
初始化配置、活动模块、Service。
2 HTTPserver/http/http.go
创建 HTTP Server，注册插件。
3 ROUTERserver/http/router.go
分发到 long、game、dzx、lhy、ccy。
4 ACTact/.../route.go
活动内注册具体接口。
前台接口

通常是 H5 调用的接口，例如活动首页数据、任务进度、榜单、领奖。常见前缀类似 /api/act。

后台接口

通常用于运营后台、内部工具、配置管理，可能会走登录态或 SSO 校验。常见前缀类似 /api/internal。

读路由时先看全局挂载，再看活动自己的 route.go23 / 24
提醒：不同活动文件名可能是 route.go 或 router.go，重点找 InitRoute。前台/后台不是语法区别，而是使用人群和权限区别。
New Activity新增活动流程

新开发一个活动，按这条线走

STEP 1看需求和 docs，确认活动类型：任务、榜单、互动、商城、Icon、Kafka、Cron。
STEP 2用脚手架生成活动骨架，优先走 create_activity_menu.sh。
STEP 3补活动代码：route、service、model、backend/internal、配置读取。
STEP 4挂全局路由、补环境 JSON，必要时挂 Cron/Kafka，最后联调验收。
给前端同学的联调清单

确认 H5 地址、接口路径、请求参数、返回 JSON 字段、错误码、活动时间、测试环境配置是否一致。提测前按“过滤 → 计算 → 发奖 → 日志”自检。

开发新活动的核心：模板复用 + 配置齐全 + 路由挂载24 / 24
这一页可以作为实战收尾：让大家知道自己接活动需求时，不是从零写 Go，而是找模板、生成骨架、补业务、联调接口。
