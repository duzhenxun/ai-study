Oh My!
三大 AI 编排插件

为 Claude Code、Codex CLI、OpenCode 打造的多智能体编排层 — 
让 AI Agent 从单兵作战升级为军团协作

oh-my-claudecode oh-my-codex oh-my-openagent
⚡

它们解决什么问题？

AI 编程助手很强，但单次会话、单一模型、线性执行 — 面对复杂工程仍然不够。这三个插件把「单个 AI」变成「一支 AI 团队」。

🎯

多智能体并行

不再一个 Agent 从头做到尾。规划、执行、审查、测试 — 多个专业 Agent 并行工作。

🧠

智能模型路由

简单任务用轻量模型，复杂推理用顶级模型。成本节省 30~50%。

🔄

自动验证循环

Ralph Loop / Team Verify — 达不到 100% 完成度绝不停止，自动修复。

🔌

零学习曲线

装好就能用，自然语言交互。无需记忆命令，只需描述需求。

For Claude Code

oh-my-claudecode

Claude Code 的多智能体编排系统。Team 模式是标准入口，stage 化流水线驱动团队协作。

👥Team 模式
team-plan → prd → exec → verify → fix 循环
🖥️omc-teams
tmux 中启动 Codex / Gemini CLI 工作者
🤝/ccg 三模型并行
Codex + Gemini + Claude 协同编排
🧩32 个专业智能体
架构、研究、设计、测试、数据科学…
📊HUD 状态栏
实时显示编排指标与智能体状态
🎓技能学习
从会话提取可复用模式，自动注入
💬通知集成
Telegram / Discord / Slack 回调
💰智能模型路由
节省 30-50% token 消耗
% /team 3:executor "fix all TypeScript errors"
